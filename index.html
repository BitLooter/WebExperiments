A couple notes about the content

As I'm sure should be obvious, this site is intented to demonstrate aspects of certain bleeding-edge web technologies. Naturally, this will sometimes require a bleeding-edge browser. This being a hobby site built to my whims, I have chosen not to actively maintain compatibility with older browsers. Not only would doing so require a greater investment of time on my part for little benefit, it would be fairly pointless anyways as many of the pages within would either not work or display an error telling you to upgrade your browser to something written this millennium. It should work properly on all modern browsers, except perhaps some of the more experimental tests; during development I've been testing against Chrome 14, Firefox 5, and occasionally Opera 11. Slightly older versions of the browsers (e.g. Firefox 3.6) will probably run most of the tests. IE is a different story; it's only fairly recently that Microsoft has displayed a serious interest in web standards and started playing catch-up to the other browsers. IE9 is the latest version as this is written, and is fairly compatible. IE8 and below are missing some important features and I won't make any guarantees about its performance. IE10 is shaping up to be very compatible, and when it is released I imagine it should handle most (all?) of the demos with minimal problems.

On a more technical note, if you go browsing through the source code, you'll notice embedded CSS and Javascript in parts. I realize this is bad form, but I've made some concessions in the name of simplicity. This is not and will not become an especially large site, and there's no reason to split every page into twenty parts just for a small collection of static HTML files. That said, duplicate and common code is shuffled off into modules whenever possible, for ease of reuse; no reason to make things difficult.
